导弹拦截
算法 2022/4/28

题目链接：https://www.acwing.com/problem/content/1012/

某国为了防御敌国的导弹袭击，发展出一种导弹拦截系统。

但是这种导弹拦截系统有一个缺陷：虽然它的第一发炮弹能够到达任意的高度，但是以后每一发炮弹都不能高于前一发的高度。

某天，雷达捕捉到敌国的导弹来袭。

由于该系统还在试用阶段，所以只有一套系统，因此有可能不能拦截所有的导弹。

输入导弹依次飞来的高度（雷达给出的高度数据是不大于30000的正整数，导弹数不超过1000），计算这套系统最多能拦截多少导弹，如果要拦截所有导弹最少要配备多少套这种导弹拦截系统。

输入格式

共一行，输入导弹依次飞来的高度。

输出格式

第一行包含一个整数，表示最多能拦截的导弹数。

第二行包含一个整数，表示要拦截所有导弹最少要配备的系统数。

数据范围

雷达给出的高度数据是不大于 $30000$ 的正整数，导弹数不超过 $1000$。

输入样例：

389 207 155 300 299 170 158 65

输出样例：

6
2

注意：贪心解与最优解并不是等价的，它们是对应着两种不同的方案的解
本题难点在于第二问的贪心解，由于第一问的最长上升子序列，很容易想到第二问的“最优解”就是找能被几个最长上升子序列所覆盖
但是“寻找一个序列最少能被几个最长上升子序列覆盖”这个问题并不好做。我们可以换思路，观察有没有别的解法。
一般来说贪心解法都可以被优先考虑
对于样例：389 207 155 300 299 170 158 65
最优解为：子序列1：389 300 299 170 158 65 子序列2: 207 155
贪心解为：子序列1 ：389 207 155 子序列2：300 299 170 158 65
通过调整法可以证明，经过交换，贪心解一定不会差于最优解，所以可以采用贪心来求解

#include <iostream>
#include <cstring>
#include <algorithm>
using namespace std;
const int N = 1010;
int a[N];
int f[N],g[N];
int n,x;
int main()
{
    while(cin >> x) a[++ n] = x;
    int res = 0;
    for(int i = 1;i <= n;i ++)
    {
        f[i] = 1;
        for(int j = 1;j < i;j ++)
        {
            if(a[i] <= a[j]) f[i] = max(f[i],f[j] + 1);
        }
        res = max(res,f[i]);
    }
    cout << res << endl;
    //贪心解为，对于一个数ai，把它放在已有的“子序列的结尾数大于ai且最小的那一个子序列中去”
    //最优解并不一定是贪心解，但是可以证明，贪心解不会差于最优解
    //所有子序列结尾的数有一个性质，它们是单调递增的
    int cnt = 0;//g从随着cnt从0开始存储，存储范围为[0,cnt - 1]
    for(int i = 1;i <= n;i ++)
    {
        int k = 0;
        //最终结尾的数要比a[i]大，所以如果小于它就一直做
        while(k < cnt && g[k] < a[i]) k ++;//从头找到第一个a[i]大于等于子序列结尾的数，因为单调递增，所以一定是最小的
        g[k] = a[i];//把它换掉
        if(k >= cnt) cnt ++;//g存储的范围在0 ~ cnt-1，如果k==cnt，就说明没有大于a[i]的数了，那么超过了所有的子序列结尾都没有匹配的，就自己创建一个子序列
    }
    printf("%d",cnt);
    return 0;    
}

玩游戏
算法 2022/4/16

题目链接：https://www.acwing.com/problem/content/4403/

$n$ 个小朋友围成一圈，玩数数游戏。

小朋友们按顺时针顺序，依次编号为 $1 \sim n$。

初始时，$1$ 号小朋友被指定为领头人。

游戏一共会行进 $k$ 轮。

在第 $i$ 轮中，领头人会从他的顺时针方向的下一个人开始，按顺时针顺序数 $a_i$ 个人。

其中，最后一个被领头人数到的人被淘汰出局，这也意味着该轮游戏结束。

出局者的顺时针方向的下一个人被指定为新领头人，引领新一轮游戏。

例如，假设当游戏即将开始第 $i$ 轮时，还剩下 $5$ 个小朋友，编号按顺时针顺序依次为 $8,10,13,14,16$，并且当前领头人为 $13$ 号小朋友，$a_i = 12$，则第 $i$ 轮游戏结束后，最后一个被数到的小朋友为 $16$ 号小朋友，他将被淘汰出局，并且处于其下一位的第 $8$ 号小朋友将被指定为新领头人。

现在，请你求出每一轮次被淘汰的小朋友的编号。

输入格式

第一行包含两个整数 $n,k$。

第二行包含 $k$ 个整数 $a_1,a_2,…,a_k$。

输出格式

一行，$k$ 个整数，其中第 $i$ 个整数表示在第 $i$ 轮中被淘汰的小朋友的编号。

数据范围

前三个测试点满足 $2 \le n \le 10$。
所有测试点满足 $2 \le n \le 100$，$1 \le k \le n-1$，$1 \le a_i \le 10^9$。

输入样例1：

7 5
10 4 11 4 1

输出样例1：

4 2 5 6 1

输入样例2：

3 2
2 5

输出样例2：

3 2

约瑟夫环问题

//约瑟夫环问题，可以用队列解决，将数过的数放在加入到队尾，这样队头就是leader，然后从leader开始数
//给定一个a的长度，每次有效长度只是a % size()
#include <iostream>
#include <algorithm>
#include <queue>
using namespace std;
const int N = 110;
int n,k;
queue<int> q;

int main()
{
    cin >> n >> k;
    for(int i = 1;i <= n;i ++) q.push(i);
    for(int i = 1;i <= k;i ++)
    {
        int a;
        cin >> a;
        a %= q.size();
        for(int j = 1;j <= a;j ++)
        {
            int t = q.front();
            q.push(t);
            q.pop();
        }
        printf("%d ",q.front());
        q.pop();
    }
    return 0;
}

ICS-1-C语言机制
计算机知识 2022/4/14

C程序执行的两个视角

静态: C代码的连续一段 总能对应到 一段连续的机器指令
动态： C代码执行的状态总能对应到 机器的状态
●源代码视角 

◾函数、变量、指针...

●机器指令视角

◾ 寄存器、内存、地址...

两个视角共同之处： 内存
◾ 代码、变量（源代码视角） = 地址 + 长度（机器指令视角）
◾ 内存 = 代码 + 数据 + 堆栈
因此理解C程序执行最重要的就是内存模型

C语言中一切皆是地址
指针都是纯粹的，类型声明只不过是对一段内存的解读方式 

#include <stdio.h>
void printptr(void *p)//空指针是非常纯粹的，它就是一个纯粹的地址，强制类型转换更改了内存的解读方式
{
    printf("*p = %p       *p = %016lx\n",p,*(long *)p);//对于p来说，首先它是一个指针，剩下的那个long代表对指针指向的内存的一个解读
}
int x;
int main(int argc,char *argv[])
{
    printptr(main);//代码
    printptr(&main);
    printptr(&x);//数据
    printptr(&argc);//堆栈
    printptr(argv);
    printptr(&argv);
    printptr(argv[0]);
}

C Type System
类型：对一段内存的解读方式。C里所有的数据都可以理解成是地址（指针）+ 类型（对地址的解读） 

#include <stdio.h>
#include <assert.h>
int main(int argc,char *argv[])
{
    int (*f)(int,char *[]) = main;
    ////利用f可以直接调用函数，刚才上面看过了它能够执行的原因，利用一个指针就可以直接把main的代码读出来
    if(argc != 0)
    {
        char ***a = &argv, *first = argv[0], ch = argv[0][0];
        printf("arg = \"%s\";    ch = '%c'\n",first, ch);//输出结果：arg = "./a.out";    ch = '.'
        assert(***a == ch);
        /*
        assert(***a == ch)等价于
        if(***a == ch)
        {
            
        }
        else
        {
            printf("ERROR\n");
            exit(1);
        }如果程序出现bug能够直接中断
        */
        f(argc - 1,argv + 1);
    }
}

C语言机制使得代码可读并且可维护
算法 2022/4/14


1、软件里有很多隐藏的dependence（依赖关系），有一些额外的，代码中没有体现和约束的“规则”
2、往往如果增加一个需求，就需要改动很多地方
一个例子是如果要模拟一个寄存器的工作，现在要加一个新的寄存器，就需要更改：X定义的位置、X1定义的位置、计算模拟的位置、写回的位置。如果这四个位置分布在一篇代码里还好，如果每个位置都分布在不同的文件中那就麻烦了

● 一处改了，另外一处就忘了(例如数组中加了一个寄存器忘记更新NREG...)
● 减少依赖关系 -> 降低代码耦合程度


好处在于只需要定义一次列表就可以对列表进行不同的处理，这种形式很利于维护
不至于把同一段代码copy很多次，很大程度上促进了代码的复用

#include <stdint.h>
#define NREG 4

typedef uint8_t u8;//定义一个8bit的int
u8 pc = 0,R[NREG];
/*
小知识：
还存在一个intptr_t这种类型，保证了int里面能够容纳指针
对于(int)ptr这种操作是非常危险的，如果在32位机里面ptr是4字节，int也是4字节那还好
但是如果把同样一段代码移植到了64位机里面int是4字节，而ptr是8字节，此时如果将指针cast成int的话那就JJ
*/

int main()
{


    //方法1
    u8 R[] = {
        0,0,0,0,0......init_pc,
    };//这种定义不好维护，很容易就漏写0，而且如果需求有更改（要再多加一个寄存器）比较麻烦


    //方法2
    enum {RA,R1 ..... PC};
    u8 R[] = {
        [RA] = 0,
        [R1] = 0,
        ...
        [PC] = init_pc,
    }//这种定义的手法，给予了代码很好的可读性，使得代码“不言自明”，而且如果要多加一个寄存器的话也只需要改列表，然后多加一个就好了

    return 0;
}
