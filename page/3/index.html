无线网络
2022/6/5

题目链接：https://www.acwing.com/problem/content/4432/

农夫约翰的农场可以看作一个二维平面。

农场中散布着 $n$ 头奶牛，每头奶牛的位置坐标已知。

农场中还建有 $2$ 个 wifi 基站，每个基站的位置坐标已知。

这 $n+2$ 个位置坐标两两不同。

第一个基站的有效覆盖范围 $r_1$ 和第二个基站的有效覆盖范围 $r_2$ 均可由约翰自由设定。

因为奶牛喜欢保持电子邮件联系，所以约翰希望所有奶牛都能被无线网络覆盖。

如果一头奶牛满足以下两个条件中的至少一个：

  • 它到第一个基站的距离不超过 $r_1$
  • 它到第二个基站的距离不超过 $r_2$

那么就视为它已被无线网络覆盖。

同时为了降低成本，约翰希望 ${r_1}^2+{r_2}^2$ 尽可能小。

请你计算 ${r_1}^2+{r_2}^2$ 的最小可能值。

输入格式

第一行包含 $5$ 个整数 $n,x_1,y_1,x_2,y_2$，其中 $n$ 为奶牛数量，$(x_1,y_1)$ 为第一个基站的坐标，$(x_2,y_2)$ 为第二个基站的坐标。

接下来 $n$ 行，每行包含两个整数 $x_i,y_i$，表示一头奶牛的位置坐标 $(x_i,y_i)$。

输出格式

输出 ${r_1}^2+{r_2}^2$ 的最小可能值，答案四舍五入到个位。

数据范围

前 $4$ 个测试点满足 $1 \le n \le 10$。
所有测试点满足 $1 \le n \le 2000$，$-10^7 \le x_i,y_i \le 10^7$。

输入样例1：

2 -1 0 5 3
0 2
5 2

输出样例1：

6

输入样例2：

4 0 0 5 0
9 4
8 3
-1 0
1 4

输出样例2：

33

这题还是挺难想的…..朴素的想法是枚举一下r1，枚举一下r2，然后再循环一遍看是否所有点都被覆盖。这是$O(n^3)$(TLE)
但是我们可以发现：1、圆的边界一定是至少有一个点的(假设没有，那么我们可以缩小圆的面积而不会使得结果变差)
2、我们可以通过更改枚举顺序进行优化。我们从大到小枚举r1距离，“从大到小保证了，每次只会从r1覆盖的点集中一次只抛出一个点，而在r2的点集中只加入一个点”。这样我们就可以在线性的时间内完成枚举所有r1和r2的选法
顺带一提这题不需要用堆进行优化，因为它在处理集合的最值过程中，对于一个集合只有“加入”而没有“删除”操作，所以用一个变量动态维护一下就可以了

#include <iostream>
#include <cstring>
#include <algorithm>
using namespace std;
#define x first
#define y second
const int N = 2010;
typedef pair<int, int> PII;
typedef long long LL;
int n;
PII s1,s2;
PII q[N];
LL get_dist(PII a,PII b)
{
    LL dx = a.x - b.x,dy = a.y - b.y;
    return dx * dx + dy * dy;
}
bool cmp(PII a,PII b)
{
    return get_dist(a,s1) < get_dist(b,s1);  
}
int main()
{
    cin >> n >> s1.x >> s1.y >> s2.x >> s2.y;
    for(int i = 0;i < n;i ++) cin >> q[i].x >> q[i].y;
    
    sort(q,q + n,cmp);
    LL res = 1e18,r = 0;
    for(int i = n - 1;i >= 0;i --)
    {
        res = min(res,r + get_dist(q[i],s1));
        r = max(r,get_dist(q[i],s2));
    }
    res = min(res,r);
    cout << res;
    return 0;
}

ICS-1-C语言机制
2022/4/14

C程序执行的两个视角

静态: C代码的连续一段 总能对应到 一段连续的机器指令
动态： C代码执行的状态总能对应到 机器的状态
●源代码视角 

◾函数、变量、指针...

●机器指令视角

◾ 寄存器、内存、地址...

两个视角共同之处： 内存
◾ 代码、变量（源代码视角） = 地址 + 长度（机器指令视角）
◾ 内存 = 代码 + 数据 + 堆栈
因此理解C程序执行最重要的就是内存模型

C语言中一切皆是地址
指针都是纯粹的，类型声明只不过是对一段内存的解读方式 

#include <stdio.h>
void printptr(void *p)//空指针是非常纯粹的，它就是一个纯粹的地址，强制类型转换更改了内存的解读方式
{
    printf("*p = %p       *p = %016lx\n",p,*(long *)p);//对于p来说，首先它是一个指针，剩下的那个long代表对指针指向的内存的一个解读
}
int x;
int main(int argc,char *argv[])
{
    printptr(main);//代码
    printptr(&main);
    printptr(&x);//数据
    printptr(&argc);//堆栈
    printptr(argv);
    printptr(&argv);
    printptr(argv[0]);
}

可以看出数据的存放位置是比代码还要高的，而且数据和代码距离堆栈都有一定的距离

惊奇的可以发现当我们用一个指针指向代码之后，它就真的把那个二进制代码直接读出来了

C Type System
类型：对一段内存的解读方式。C里所有的数据都可以理解成是地址（指针）+ 类型（对地址的解读） 

#include <stdio.h>
#include <assert.h>
int main(int argc,char *argv[])
{
    int (*f)(int,char *[]) = main;
    ////利用f可以直接调用函数，刚才上面看过了它能够执行的原因，利用一个指针就可以直接把main的代码读出来
    if(argc != 0)
    {
        char ***a = &argv, *first = argv[0], ch = argv[0][0];
        printf("arg = \"%s\";    ch = '%c'\n",first, ch);//输出结果：arg = "./a.out";    ch = '.'
        assert(***a == ch);
        /*
        assert(***a == ch)等价于
        if(***a == ch)
        {
            
        }
        else
        {
            printf("ERROR\n");
            exit(1);
        }如果程序出现bug能够直接中断
        */
        f(argc - 1,argv + 1);
    }
}

C语言机制使得代码可读并且可维护
2022/4/14


1、软件里有很多隐藏的dependence（依赖关系），有一些额外的，代码中没有体现和约束的“规则”
2、往往如果增加一个需求，就需要改动很多地方
一个例子是如果要模拟一个寄存器的工作，现在要加一个新的寄存器，就需要更改：X定义的位置、X1定义的位置、计算模拟的位置、写回的位置。如果这四个位置分布在一篇代码里还好，如果每个位置都分布在不同的文件中那就麻烦了

● 一处改了，另外一处就忘了(例如数组中加了一个寄存器忘记更新NREG...)
● 减少依赖关系 -> 降低代码耦合程度


好处在于只需要定义一次列表就可以对列表进行不同的处理，这种形式很利于维护
不至于把同一段代码copy很多次，很大程度上促进了代码的复用

#include <stdint.h>
#define NREG 4

typedef uint8_t u8;//定义一个8bit的int
u8 pc = 0,R[NREG];
/*
小知识：
还存在一个intptr_t这种类型，保证了int里面能够容纳指针
对于(int)ptr这种操作是非常危险的，如果在32位机里面ptr是4字节，int也是4字节那还好
但是如果把同样一段代码移植到了64位机里面int是4字节，而ptr是8字节，此时如果将指针cast成int的话那就JJ
*/

int main()
{


    //方法1
    u8 R[] = {
        0,0,0,0,0......init_pc,
    };//这种定义不好维护，很容易就漏写0，而且如果需求有更改（要再多加一个寄存器）比较麻烦


    //方法2
    enum {RA,R1 ..... PC};
    u8 R[] = {
        [RA] = 0,
        [R1] = 0,
        ...
        [PC] = init_pc,
    }//这种定义的手法，给予了代码很好的可读性，使得代码“不言自明”，而且如果要多加一个寄存器的话也只需要改列表，然后多加一个就好了

    return 0;
}
